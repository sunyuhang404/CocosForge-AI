# 连连看游戏策划案

## 游戏目标
玩家需要在规定的时间内，通过消除相同的图标来获得分数。每一关卡都有一个目标分数，达到目标分数即可过关。游戏共有多个关卡，每个关卡的难度逐渐增加。

## 核心玩法循环
1. 游戏开始时，随机生成一个包含不同图标的棋盘。
2. 玩家使用鼠标点击两个相同的图标，如果这两个图标之间可以通过不多于3个直角转弯的路径相连，则消除这两个图标，并获得一定的分数。
3. 每次成功消除图标后，上方的图标会下落填补空缺的位置，同时可能会触发连锁反应，继续消除图标。
4. 当棋盘上没有可以消除的图标时，游戏结束或者进入下一关卡（根据当前关卡是否完成目标分数）。
5. 玩家可以使用一些道具来帮助消除图标，例如重新排列图标、延长游戏时间等。

## 关卡与难度
- 初始关卡：棋盘大小为6x6，目标分数为100分。
- 随着关卡的递增，棋盘大小和目标分数逐渐增加。
- 棋盘大小最大不超过20x20，以保证游戏的可玩性和挑战性。
- 每个关卡的难度可以通过增加图标种类、减少游戏时间等方式进行调整。

## 输入控制
- 玩家使用鼠标点击棋盘上的图标进行选择。
- 选中第一个图标后，再次点击另一个相同图标进行匹配。
- 可以通过点击道具按钮来使用道具。

## 资源清单
- 图标资源：包括各种不同的图标图片。
- 道具资源：包括重新排列图标、延长游戏时间等道具的图标和效果。
- 背景音乐和音效：用于增加游戏氛围。
- UI界面资源：包括游戏界面、得分板、计时器等UI元素。

## 开发里程碑
1. **需求分析与设计 (1周)**
   - 完成游戏策划案的编写。
   - 设计游戏UI界面。
   - 确定图标和道具的设计风格。

2. **核心功能开发 (2周)**
   - 实现棋盘生成和图标随机分布逻辑。
   - 实现图标消除和下落填补空缺的逻辑。
   - 实现游戏计分和计时功能。
   - 实现基本的输入控制和道具使用逻辑。

3. **关卡设计与测试 (1周)**
   - 设计多个关卡，每个关卡有不同的棋盘大小和目标分数。
   - 测试游戏的稳定性和可玩性。
   - 优化游戏体验，调整关卡难度。

4. **美术资源制作 (1周)**
   - 制作图标和道具的美术资源。
   - 制作背景音乐和音效。
   - 制作UI界面资源。

5. **集成与调试 (1周)**
   - 将所有资源集成到游戏中。
   - 调试游戏，修复可能存在的bug。
   - 优化游戏性能和用户体验。

6. **发布与推广 (1周)**
   - 准备游戏发布所需的材料，如宣传图、游戏说明等。
   - 发布游戏到各大平台。
   - 进行游戏推广，吸引玩家下载和体验。
